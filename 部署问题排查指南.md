# 部署问题排查指南

## 2026-07-02: 英语错题分析上传多张图片失败（413/504错误）

### 核心问题
- **nginx限制了上传大小**：默认 `client_max_body_size` 为1MB
- **nginx超时设置太短**：默认60秒，AI分析多张图片需要更长时间
- **与代码修改无关**：是nginx配置问题，不是Next.js或API代码问题

### 错误表现
- 单张图片：正常分析
- 多张图片：413 Request Entity Too Large（上传大小限制）
- 修复413后：504 Gateway Time-out（超时限制）

### 解决方案
1. 检查nginx配置文件位置：`/etc/nginx/conf.d/cc-mom-website.conf`
2. 添加上传大小限制：在 `location /` 块中添加 `client_max_body_size 20M;`
3. 添加超时设置：在 `location /` 块中添加：
   ```nginx
   proxy_connect_timeout 300s;
   proxy_send_timeout 300s;
   proxy_read_timeout 300s;
   ```
4. 测试并重启nginx：`nginx -t && nginx -s reload`

### 完整配置示例
```nginx
location / {
    proxy_pass http://localhost:3000;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection 'upgrade';
    proxy_set_header Host $host;
    proxy_cache_bypass $http_upgrade;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    client_max_body_size 20M;
    proxy_connect_timeout 300s;
    proxy_send_timeout 300s;
    proxy_read_timeout 300s;
}
```

### 快速诊断
```bash
# 检查nginx是否运行
ps aux | grep nginx

# 检查nginx配置文件
cat /etc/nginx/conf.d/cc-mom-website.conf

# 检查上传限制
cat /etc/nginx/conf.d/cc-mom-website.conf | grep client_max_body_size

# 检查超时设置
cat /etc/nginx/conf.d/cc-mom-website.conf | grep timeout

# 测试nginx配置
nginx -t

# 重启nginx
nginx -s reload
```

## 常见错误和解决方案

### "Failed to find Server Action"
原因：浏览器缓存了旧版本代码
解决：`Ctrl+Shift+R` 强制刷新

### "git pull aborted" 因本地修改
原因：服务器上有未提交的修改
解决：
```bash
git reset --hard origin/main  # 丢弃本地修改
git pull origin main          # 重新拉取
```

### API返回401 Unauthorized
1. 检查sessionStorage是否保存了密码
2. 检查服务器API代码密码验证逻辑
3. 确认环境变量是否正确配置

### heredoc命令卡住（出现 > 提示符）
原因：EOF标记不识别
解决：改用 echo 命令逐行创建文件
```bash
echo 'line1' > file.txt
echo 'line2' >> file.txt
```
