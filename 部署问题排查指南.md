## 快速定位401/认证问题

### 第1步：检查浏览器Network
```
F12 → Network → 找到失败的API请求 → 查看状态码
```
- 401 = 密码/认证问题
- 500 = 服务器代码错误

### 第2步：检查服务器日志
```bash
pm2 logs --lines 50
```

### 第3步：检查服务器实际文件内容（最关键）
```bash
cat /var/www/cc-mom-website/app/api/xxx/route.ts | grep "关键代码"
```
**确认服务器代码是否真的更新了**

### 第4步：检查sessionStorage（浏览器控制台）
```javascript
console.log(sessionStorage.getItem('admin_password'))
```

### 第5步：强制更新服务器代码
```bash
cd /var/www/cc-mom-website
git fetch origin
git reset --hard origin/main
rm -rf .next
npm run build
pm2 restart all
```

### 第6步：强制刷新浏览器
```
Ctrl + Shift + R
```

## 常见错误和解决方案

### "Failed to find Server Action"
原因：浏览器缓存了旧版本代码
解决：`Ctrl+Shift+R` 强制刷新

### "git pull aborted" 因本地修改
原因：服务器上有未提交的修改
解决：
```bash
git reset --hard origin/main  # 丢弃本地修改
git pull origin main          # 重新拉取
```

### API返回401 Unauthorized
1. 检查sessionStorage是否保存了密码
2. 检查服务器API代码密码验证逻辑
3. 确认环境变量是否正确配置

### heredoc命令卡住（出现 > 提示符）
原因：EOF标记不识别
解决：改用 echo 命令逐行创建文件
```bash
echo 'line1' > file.txt
echo 'line2' >> file.txt
```
